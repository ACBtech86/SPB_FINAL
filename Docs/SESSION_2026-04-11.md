## Database correlation key

The link between request and response is **MQ-level**, not XML:

```
received.mq_correl_id == sent.mq_msg_id
```

Example pair I verified in the DB:
- Sent (`spb_local_to_bacen`) — `db_datetime=2026-04-11 17:47:09.819257`, `cod_msg=GEN0001`, `mq_msg_id=414d5120514d2e33363236363735312e6f1fd069032d0040`
- Received (`spb_bacen_to_local`) — `db_datetime=2026-04-11 17:47:10.002053`, `cod_msg=GEN0001R1`, `mq_correl_id=414d5120514d2e33363236363735312e6f1fd069032d0040` ← matches sent.mq_msg_id

The `nu_ope` field does **not** match across request/response — BACEN (and the auto-responder) generates its own `nu_ope` for the response. Don't try to correlate by `nu_ope`.
